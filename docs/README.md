📄 Project Documentation
This folder contains the formal technical documentation, academic reports, and design analysis for the Morse Code Temperature Encoder project.

📂 Document Overview
1. Final Technical Report
The comprehensive final document covering the entire project lifecycle.

Contents: System architecture, hardware/software integration details, flowcharts, and final testing results.

Key Sections: Technical specifications of the ATmega328P, Morse algorithm implementation, and Wheatstone bridge calculations.

2. Analysis & Design Study
A detailed study of the components and theoretical foundations.

Contents: Market research for component selection (PT100, Buzzer, LEDs), cost analysis, and comparison between SMD vs. THT components for the USB interface.

3. Project Scope & Problem Statement
The initial planning phase of the project.

Contents: Definition of functional requirements (Mode 1: Temperature; Mode 2: Manual Serial Input), non-functional requirements (Baud rate, safety protocols), and project milestones.

4. Hardware Diagrams
Early-stage design concepts and block diagrams.

Contents: High-level system overview showing the interaction between the power protection circuit, the MCU, and the I/O peripherals.

🎓 Academic Context
Course: Laboratory of Hardware Development

Institution: IPCA - Escola Técnica Superior Profissional (Portugal)

Objective: To design a complete embedded system from scratch, including PCB manufacturing and firmware development in C.
